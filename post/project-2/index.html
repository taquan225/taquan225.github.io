Book Recommender System

June 21, 2022

I built a content-based recommender system. Based on the author, the tags, and the rating of one book, I can recommend the users with books that they may like. I am using the vector space model and cosine distance between books’ vectors to find similar books. In this case, the vectors I are using are author_vector and tags_vector.

For the dataset, the code and further description: Link to GitHub Repository

Methods

  • Preprocessing the Data:
    • Merge the book_tags and tags dataset to generate the final book_tags dataset. This dataset contains book_id, tag_id, and count
    • Drop unnecessary columns and kept only: book_id, original_publication_year, original_title, language_code, average_rating, authors, book_count
    • Drop rows with null values. By doing this, I reduce the number of rows from 10,000 to 8,405
    • Generate author_vector and tag_vector columns based on the book_tags data frame and column authors in the books data frame
  • Content-based Recommendation System:
    • get_rated_books():
      • Input: user_id
      • Output: the list of book
      • In this function, I are rating the book if they are greater than 4 by using user_id as an input.
    • cal_distance():
      • Input: 2 book_id and data frame
      • Output: The distance
      • In this function, I are calculating the cosine distance betIen book1 to book2 to get the target one from the dataset based on author_vector and tags_vector.
    • top_k_neighbor() :
      • Input: book_id, list of user_id’s favorite book, data frame
      • Output: list of similar book_id and distance
      • I am using the top_k_neighbor function to find the nearest neighbor of the book_id I provide. K is the number of neighbors that I need to find.
    • recommend_book():
      • Input: user_id, number of books the user want to have
      • Output: The name of those books that I recommend to users
      • This function takes the user_id and an integer n as inputs. The output is n books that are the most similar to those the user likes.

Interpret the recommended results:

After running this function I can see the 8 books that are similar to those that are rated over 4 by the user. In this case, I test to recommend 8 books for user_id 9.

The recommender system manages to recommend 8 books for the users and prints out the names of those books.

Evaluate your system’s performance:

I evaluated the system based on

  • Accuracy: I use Root Mean Square Error (RMSE). The accuracy of 0.8466729436823051 is based on all users’ ratings. From this accuracy, I can say this dataset is the best fit for our recommended systems.
  • Coverage: I did not cover books that are rated loIr than 4, as Ill as books that contain null values.
  • Novelty: I are likely to recommend books that the user may not aware of their existence. It is due to the tags are taken into account of the recommender system.

Limitation and future improvement of your system:

  • The running time of the system is long. It is due to the number of books I take from the users to recommend from is high.
    • To improve on this, I may narrow down the number of books I take from the user down to only five or three. I can do this by changing the score threshold to be 4.5. Other than this, I can change the fixed number of books I take from the user to be five with the highest ratings. By doing this, I can reduce the training time.
  • Some tags and authors’ names are not in English. It reduces the number of books I can recommend to users, as I cannot access those books’ tags that are not in English.
    • To improve on this, I can apply a text classification to deal with characters that are not English. As I can read these words, I can improve our system by a huge margin. Users can have access to books that are similar in content but are not titled in English.
  • I did not take many features of books into consideration. Therefore, our recommendation results may not be the best.
    • To improve on this, I would want to take the published years into account. As books from different generations tend to be different, users tend to prefer books at certain times. Therefore, the recommender system can take year into account to provide better books for users
